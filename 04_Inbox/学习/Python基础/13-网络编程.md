# 网络编程

## 网络三要素

- **IP 地址**：标识网络中的设备
- **端口**：标识设备上的应用程序
- **协议**：通信规则（TCP、UDP、HTTP 等）

## TCP/IP 协议族

四层模型（从上到下）：

| 层级 | 功能 | 常见协议 |
|------|------|----------|
| 应用层 | 为应用提供服务 | HTTP、FTP、DNS、SMTP |
| 传输层 | 端到端通信 | TCP、UDP |
| 网络层 | 路由寻址 | IP、ICMP |
| 链路层 | 物理传输 | Ethernet、WiFi |

## IP 地址

- **IPv4**：32 位，如 `192.168.1.1`，约 43 亿个地址
- **IPv6**：128 位，如 `2001:0db8::1`，几乎无限
- **子网掩码**：区分网络号和主机号，如 `255.255.255.0`
- **公网 IP**：互联网上唯一标识
- **私网 IP**：局域网内使用（10.x.x.x / 172.16-31.x.x / 192.168.x.x）

```python
import socket
# 获取本机主机名和IP
hostname = socket.gethostname()
ip = socket.gethostbyname(hostname)
print(f"{hostname}: {ip}")
```

## 端口

- **公认端口**（0-1023）：系统服务占用，如 80(HTTP)、443(HTTPS)、22(SSH)
- **动态端口**（1024-65535）：用户程序使用

## Socket 套接字

Socket 是网络通信的端点，是对 TCP/IP 协议的封装。

```python
import socket

# 创建 TCP socket
tcp_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 创建 UDP socket
udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
```

## UDP 编程

无连接、不可靠、速度快，适合实时性要求高的场景（视频、游戏）。

```python
import socket

# === 发送端 ===
udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
udp_socket.sendto("Hello".encode(), ("127.0.0.1", 8888))
udp_socket.close()

# === 接收端 ===
udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
udp_socket.bind(("", 8888))
data, addr = udp_socket.recvfrom(1024)
print(f"收到来自 {addr} 的数据: {data.decode()}")
udp_socket.close()
```

## TCP 编程

面向连接、可靠传输、三次握手建立连接、四次挥手断开连接。

### 服务端

```python
import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(("0.0.0.0", 9999))
server.listen(5)  # 最大等待连接数
print("服务器启动，等待连接...")

conn, addr = server.accept()  # 阻塞等待客户端连接
print(f"客户端 {addr} 已连接")

data = conn.recv(1024)
print(f"收到: {data.decode()}")
conn.send("收到你的消息".encode())

conn.close()
server.close()
```

### 客户端

```python
import socket

client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client.connect(("127.0.0.1", 9999))

client.send("Hello Server".encode())
response = client.recv(1024)
print(f"服务器回复: {response.decode()}")

client.close()
```

### 三次握手与四次挥手

**三次握手（建立连接）**：
1. 客户端 → SYN → 服务端
2. 服务端 → SYN+ACK → 客户端
3. 客户端 → ACK → 服务端

**四次挥手（断开连接）**：
1. 主动方 → FIN → 被动方
2. 被动方 → ACK → 主动方
3. 被动方 → FIN → 主动方
4. 主动方 → ACK → 被动方

## HTTP 协议

### 请求消息结构

```
请求行:    GET /index.html HTTP/1.1
请求头:    Host: www.example.com
           Content-Type: application/json
空行
请求体:    {"key": "value"}  (POST 时有)
```

### 响应消息结构

```
状态行:    HTTP/1.1 200 OK
响应头:    Content-Type: text/html
           Content-Length: 1234
空行
响应体:    <html>...</html>
```

### 请求方法

| 方法 | 用途 |
|------|------|
| GET | 获取资源 |
| POST | 提交数据 |
| PUT | 更新资源 |
| DELETE | 删除资源 |
| PATCH | 部分更新 |

### 状态码

| 范围 | 含义 | 常见 |
|------|------|------|
| 1xx | 信息 | 100 Continue |
| 2xx | 成功 | 200 OK, 201 Created |
| 3xx | 重定向 | 301 永久, 302 临时, 304 未修改 |
| 4xx | 客户端错误 | 400 Bad Request, 403 Forbidden, 404 Not Found |
| 5xx | 服务端错误 | 500 Internal Error, 502 Bad Gateway |

## 实战案例

### requests 发送 HTTP 请求

```python
import requests

# GET 请求
resp = requests.get("https://httpbin.org/get", params={"name": "python"})
print(resp.status_code)  # 200
print(resp.json())

# POST 请求
resp = requests.post("https://httpbin.org/post",
                     json={"username": "admin", "password": "123"})
print(resp.json())

# 设置请求头
headers = {"User-Agent": "MyApp/1.0"}
resp = requests.get("https://httpbin.org/headers", headers=headers)
```

### Starlette 构建 Web 接口

```python
from starlette.applications import Starlette
from starlette.responses import JSONResponse
from starlette.routing import Route
import uvicorn

async def homepage(request):
    return JSONResponse({"message": "Hello, World!"})

async def user_info(request):
    user_id = request.path_params["user_id"]
    return JSONResponse({"user_id": user_id, "name": "张三"})

app = Starlette(routes=[
    Route("/", homepage),
    Route("/user/{user_id}", user_info),
])

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

---

> [!tip] 核心要点
> - UDP 快但不可靠，TCP 可靠但有连接开销
> - TCP 编程：服务端 bind→listen→accept，客户端 connect
> - HTTP 是应用层协议，基于 TCP，请求-响应模式
> - requests 库是 Python HTTP 客户端的首选
