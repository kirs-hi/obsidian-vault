# 进程与线程

## 并发与并行

- **并发（Concurrency）**：多个任务交替执行，宏观上"同时"，微观上轮流（单核 CPU）
- **并行（Parallelism）**：多个任务真正同时执行（多核 CPU）

## 多进程

### 进程的概念

进程是操作系统**资源分配**的最小单位。每个进程有独立的内存空间。

### 创建进程

```python
from multiprocessing import Process
import os

def task(name):
    print(f"子进程 {name}, PID={os.getpid()}, 父PID={os.getppid()}")

if __name__ == "__main__":
    p = Process(target=task, args=("A",))
    p.start()   # 启动进程
    p.join()    # 等待子进程结束
    print(f"主进程 PID={os.getpid()}")
```

### 自定义 Process 子类

```python
from multiprocessing import Process

class MyProcess(Process):
    def __init__(self, name):
        super().__init__()
        self.pname = name

    def run(self):
        print(f"进程 {self.pname} 正在运行")

if __name__ == "__main__":
    p = MyProcess("Worker")
    p.start()
    p.join()
```

### 进程池 Pool

管理大量进程时使用进程池，避免频繁创建销毁：

```python
from multiprocessing import Pool

def square(x):
    return x * x

if __name__ == "__main__":
    with Pool(4) as pool:
        # map：批量提交
        results = pool.map(square, range(10))
        print(results)  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

        # apply_async：异步提交单个任务
        result = pool.apply_async(square, (5,))
        print(result.get())  # 25
```

### 进程间通信

进程之间**不共享全局变量**，需要通过 Queue 通信：

```python
from multiprocessing import Process, Queue

def producer(q):
    for i in range(5):
        q.put(f"数据-{i}")

def consumer(q):
    while not q.empty():
        print(f"消费: {q.get()}")

if __name__ == "__main__":
    q = Queue()
    p1 = Process(target=producer, args=(q,))
    p2 = Process(target=consumer, args=(q,))

    p1.start()
    p1.join()
    p2.start()
    p2.join()
```

在进程池中使用 `Manager().Queue()`：

```python
from multiprocessing import Pool, Manager

def worker(q, val):
    q.put(val * 2)

if __name__ == "__main__":
    manager = Manager()
    q = manager.Queue()

    with Pool(4) as pool:
        for i in range(5):
            pool.apply_async(worker, (q, i))
        pool.close()
        pool.join()

    while not q.empty():
        print(q.get())
```

## 多线程

### 线程的概念

线程是 CPU **调度**的最小单位。同一进程内的线程共享内存空间。

### 创建线程

```python
import threading
import time

def task(name, delay):
    print(f"线程 {name} 开始")
    time.sleep(delay)
    print(f"线程 {name} 结束")

t1 = threading.Thread(target=task, args=("A", 2))
t2 = threading.Thread(target=task, args=("B", 1))

t1.start()
t2.start()
t1.join()
t2.join()
print("所有线程完成")
```

### 自定义 Thread 子类

```python
import threading

class MyThread(threading.Thread):
    def __init__(self, name):
        super().__init__()
        self.tname = name

    def run(self):
        print(f"线程 {self.tname} 正在运行")

t = MyThread("Worker")
t.start()
t.join()
```

### 线程池 ThreadPoolExecutor

```python
from concurrent.futures import ThreadPoolExecutor
import time

def download(url):
    time.sleep(1)
    return f"下载完成: {url}"

urls = ["url1", "url2", "url3", "url4"]

with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(download, url) for url in urls]
    for f in futures:
        print(f.result())
```

### 互斥锁 Lock

多线程共享数据时需要加锁保证线程安全：

```python
import threading

counter = 0
lock = threading.Lock()

def increment(n):
    global counter
    for _ in range(n):
        lock.acquire()    # 加锁
        counter += 1
        lock.release()    # 释放锁

# 更推荐用 with 语法
def increment_v2(n):
    global counter
    for _ in range(n):
        with lock:
            counter += 1

threads = [threading.Thread(target=increment, args=(100000,)) for _ in range(5)]
for t in threads:
    t.start()
for t in threads:
    t.join()

print(counter)  # 500000
```

### GIL（全局解释器锁）

CPython 中的 GIL 保证同一时刻只有一个线程执行 Python 字节码。这意味着：

- **CPU 密集型任务**：多线程无法利用多核，应使用多进程
- **IO 密集型任务**：线程在等待 IO 时会释放 GIL，多线程仍然有效

## 进程 vs 线程对比

| 对比项 | 进程 | 线程 |
|--------|------|------|
| 内存空间 | 独立 | 共享（同一进程内） |
| 创建/切换开销 | 大 | 小 |
| 通信方式 | Queue/Pipe/共享内存 | 直接访问共享变量 |
| 安全性 | 互不影响 | 需要锁机制 |
| 适用场景 | CPU 密集型 | IO 密集型 |
| 稳定性 | 一个崩溃不影响其他 | 一个崩溃可能影响整个进程 |

---

> [!tip] 核心要点
> - 进程隔离安全但开销大，线程共享高效但需加锁
> - GIL 限制了 CPython 多线程的 CPU 并行能力
> - CPU 密集 → 多进程，IO 密集 → 多线程/协程
> - 进程间通信用 Queue，线程间共享用 Lock 保护
