# 错误和异常

## 语法错误 vs 异常

- **语法错误（SyntaxError）**：代码不符合 Python 语法规则，程序无法运行
- **异常（Exception）**：语法正确但运行时出错，可以被捕获和处理

```python
# 语法错误 —— 无法运行
if True
    print("hello")

# 异常 —— 运行时出错
print(1 / 0)  # ZeroDivisionError
```

## 异常处理

### try-except

```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("除数不能为零")
```

### 捕获指定类型并获取信息

```python
try:
    num = int("abc")
except ValueError as e:
    print(f"值错误：{e}")
```

### 捕获多种异常

```python
try:
    # 可能出错的代码
    pass
except (TypeError, ValueError) as e:
    print(f"类型或值错误：{e}")
except Exception as e:
    print(f"其他错误：{e}")
```

### else 和 finally

```python
try:
    result = 10 / 2
except ZeroDivisionError:
    print("除零错误")
else:
    # 没有异常时执行
    print(f"结果是 {result}")
finally:
    # 无论是否异常都执行（常用于清理资源）
    print("执行完毕")
```

## 抛出异常

使用 `raise` 主动抛出异常：

```python
def set_age(age):
    if age < 0 or age > 150:
        raise ValueError(f"年龄不合法：{age}")
    print(f"年龄设置为 {age}")

try:
    set_age(200)
except ValueError as e:
    print(e)  # 年龄不合法：200
```

## assert 断言

调试阶段使用，条件为 False 时抛出 AssertionError：

```python
def divide(a, b):
    assert b != 0, "除数不能为零"
    return a / b

divide(10, 0)  # AssertionError: 除数不能为零
```

> 注意：断言可通过 `python -O` 运行时全局禁用，不要用于生产环境的输入校验。

## 自定义异常

继承 `Exception` 类创建自己的异常类型：

```python
class InsufficientBalanceError(Exception):
    """余额不足异常"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(f"余额不足：当前 {balance}，需要 {amount}")

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientBalanceError(balance, amount)
    return balance - amount

try:
    withdraw(100, 200)
except InsufficientBalanceError as e:
    print(e)  # 余额不足：当前 100，需要 200
```

## 异常的传递

异常会逐层向上传递，直到被 try-except 捕获。如果没有被捕获，程序终止并打印 traceback：

```python
def func1():
    print(1 / 0)  # 这里抛出异常

def func2():
    func1()  # 异常向上传递

def func3():
    try:
        func2()  # 在这里捕获
    except ZeroDivisionError:
        print("捕获到异常")

func3()  # 捕获到异常
```

## with 关键字与上下文管理器

`with` 语句利用上下文管理器协议（`__enter__` 和 `__exit__`）自动管理资源：

```python
# 自定义上下文管理器
class FileManager:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode

    def __enter__(self):
        """进入 with 块时调用，返回值赋给 as 变量"""
        self.file = open(self.filename, self.mode)
        return self.file

    def __exit__(self, exc_type, exc_val, exc_tb):
        """离开 with 块时调用（无论是否异常）"""
        self.file.close()
        return False  # 返回 True 则吞掉异常

with FileManager("test.txt", "w") as f:
    f.write("hello")
```

## Python 常见异常

| 异常 | 说明 |
|------|------|
| `TypeError` | 类型不匹配（如 `"1" + 1`） |
| `ValueError` | 值不合适（如 `int("abc")`） |
| `KeyError` | 字典中不存在的键 |
| `IndexError` | 索引超出范围 |
| `FileNotFoundError` | 文件不存在 |
| `ZeroDivisionError` | 除以零 |
| `AttributeError` | 对象没有该属性 |
| `ImportError` | 导入模块失败 |
| `NameError` | 使用未定义的变量 |
| `StopIteration` | 迭代器没有更多元素 |
| `OSError` | 系统操作错误 |
| `RuntimeError` | 一般运行时错误 |
