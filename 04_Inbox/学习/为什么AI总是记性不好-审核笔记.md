# 为什么 AI 总是「记性不好」？— 审核笔记

> 原文链接：https://km.sankuai.com/collabpage/2761659430

## 需要修正的问题

### 1. 「上下文是无限增长的」— 前后矛盾

原文："注意力是有限的，而上下文是无限增长的。"

前面刚说了上下文窗口有长度限制，这里又说"无限增长"，逻辑自相矛盾。准确说法应该是"对话内容在不断增长，直到逼近或触达上下文窗口上限"。

### 2. Lost in the Middle 推论错误（最大问题）

原文：
> 开头（强）→ 中间（弱）→ 结尾（强）
>
> 这就解释了一个你可能有过的体验：在对话中途补充的规则，反而比开头设定的规则更容易被遵守——因为它在「结尾区」，注意力权重更高。

问题：按 Lost in the Middle 的结论，开头权重本身就强，不应该比"中途补充的"更弱。作者混淆了两件事：

- "刚刚说的话"暂时处于结尾区所以权重高 — 这是一个**滑动窗口效应**，随着对话继续这条规则会滑入中间区
- "开头设定的规则"始终在开头区，按 Lost in the Middle 理论应该始终有较高权重

正确理解：开头规则的权重本身是高的，但当对话极长时，开头的位置优势也会被注意力稀释削弱；而"最近刚说过的"内容因为处于结尾区所以暂时权重高，但这只是暂时的。真正的实用结论应该是：**最重要的规则既要放在开头，也要在长对话中途定期重复**，让它同时享有开头优势和结尾优势。

### 3. 「系统提示在技术实现上有更稳定的权重保障」— 有误导

System Prompt 并没有在注意力机制层面获得额外的数学权重加成。它的"更稳定"主要来自：(1) 位置固定在最前面，享受"开头注意力强"的位置优势；(2) 部分 API 实现中 system 角色的内容在多轮对话中不会被截断。说"权重保障"容易让人误以为有特殊的技术处理，改为"位置稳定性保障"更准确。

### 4. 「注意力是有限的」— 措辞不够精确

Transformer 的 Self-Attention 是对每个 token 计算与所有其他 token 的相关性分数（Softmax 归一化），数学上等价于"竞争关系"（softmax 是零和的），所以底层逻辑成立。但"有限"不如"竞争性"精确。作为科普可接受。

### 5. 提示词漂移第一个来源的解释 — 有误导性

原文："如果某一轮对话中有一段较长的内容，模型可能会在接下来的回复中倾向于「对齐」这个长度。"

更准确的说法是：模型会从上下文中的示例（包括自己之前的回复）学习模式，如果之前的回复越来越长，模型会把这当作"正确的输出模式"来延续。这是 in-context few-shot learning 的副作用。

## 没有问题的部分

- "AI 没有记忆，只有上下文" ✓
- 上下文窗口的硬截断描述 ✓
- Attention 机制是 Transformer 核心 ✓
- AI 说"我记住了"只是文本模式输出而非真实承诺 ✓
- 实用建议（规则前置、中途重复、新任务开新对话等） ✓

## 总结

核心知识框架没有方向性错误，对读者建立正确认知有帮助。主要问题是第2点（Lost in the Middle 的推论逻辑链断了）和第3点（System Prompt 的解释过于简化导致误导），建议修正。
