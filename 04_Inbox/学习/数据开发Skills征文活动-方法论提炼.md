# 数据开发 Skills 征文活动——方法论提炼

> 来源：「数据开发 Skills 布道者」征文活动（2026.3.31—4.26），共分析 11 篇获奖文章。

## 一、原子化 + 编排层：Skill 架构设计的共识

几乎所有成功的 Skill 都遵循同一个架构范式：底层是细粒度的原子 Skill（每个控制在 500 行以内），上层是编排 Skill 负责串联流程。

- 公共数据切换用 6 个原子 Skill 串联"分析→执行→验证"三层结构
- Harness 工程用编排层→开发测试闭环→底层原子能力的三层架构
- mafka2kafka 把四步操作拆成四个独立可调用的原子能力

核心逻辑：超过 500 行 AI 容易"偷懒"跳过约束，超过 100 个字段 Token 过长会遗漏。原子化不仅是代码组织问题，更是 AI 能力边界的工程适配。

## 二、人机协作边界必须显式设计

11 篇文章无一例外都强调了"人工确认节点"：

- SLA 巡检：参数修改需人工确认
- 公共数据切换：4 个确认节点
- 数据 API 助手：3 个强制确认节点
- mafka2kafka：全程草稿不自动上线

最精辟的表述来自数据 API 助手："AI 定位为执行助手，而非自主决策者。"这不是保守，而是建立信任的唯一路径——用户敢用，才会持续用。

## 三、用机制保障质量，而非靠 Prompt 自觉

这是最有工程味道的一条方法论：

- SLA 巡检铁律："步骤间禁止 AI 判断，所有数据必须 100% 来自脚本输出"，直接杜绝幻觉
- 公共数据切换：D-001~D-008 八条双写比对"生死红线"，Wiki 规范检查用代码级强制，不通过则阻断流程
- Harness 工程：开发 Agent 和测试 Agent 上下文隔离——测试子 Agent 只读 task.md、dev_report.md 和 ETL SQL，不感知开发过程中的中间思考，打破"同一个 AI 既写代码又写测试"的自证闭环
- SaaS 全链路：Hook 体系（三级拦截 BLOCK/WARN/放行）把规范从"文档里写着"变成"代码里执行着"

## 四、CLI/接口优先，拒绝 GUI 自动化

数据 API 助手说得最直白："只要有一个环节依赖 GUI，整个链路就是脆弱的。"所有成功的 Skill 都把平台操作封装为 CLI 命令，AI 只需"写命令、看输出、处理结果"。实验迁移也明确提出架构演进方向是"从浏览器优先逐步转向接口/CLI 优先"。浏览器自动化只是过渡方案，不是终态。

## 五、知识沉淀是 Skill 越用越好的飞轮

SaaS 全链路提出了最完整的"自进化飞轮"：

1. 捕获层：Hook 自动追踪错误和用户纠正信号
2. 沉淀层：`.learnings/` 目录存储经验
3. 晋升层：每 3 天自动运行 `memory_distill.py`，对经验按 route 聚类分析，晋升为团队规范/检查规则/Hook/新 Skill

数据 API 助手用 LLM-Wiki 实现"SOP 更新即 Agent 能力更新，无需改代码"。公共数据切换强调"业务口径翻译是 AI 的硬伤，需要人工基于实际场景持续输入知识库"。

核心洞察：AI 没有记忆，所以显性化的知识存储不是锦上添花，而是 Skill 能力的地基。

## 六、状态机驱动 + 断点续作

长流程最怕中断后丢失进度：

- Harness 工程通过 `agent_state.md` 统一管理所有状态，实现断点续作
- SaaS 全链路用"文件化传递 > 对话上下文"的原则，多步骤流程产出写入文件，后续阶段读文件而非依赖对话记忆

这解决了 AI 对话上下文有限的根本问题——文件系统才是"单一事实来源"。

## 七、诚实面对 AI 的能力边界

数据交付测评是最有价值的"反面教材"——它诚实记录了 AI 的各种问题：无法识别需求文档自相矛盾、会改逻辑但不跟使用者确认、ETL 代码存在冗余维度设计、多次纠正才改对。PM 做数据开发也指出 AI 的根本局限："没有'最近做过类似事情'的感知，使用效率取决于喂给它的上下文质量。"

共识：AI 最擅长"机械性流程的串联和执行"，建模判断、需求理解、口径确认等"判断性"工作仍然需要人工把关。

## 八、渐进式落地，先拆解再逐环节覆盖

数仓变更适配的策略最务实：先把完整流程拆解为 7 个标准环节，逐一匹配 Skill，本次只落地了其中 3 个环节就已经提效 50%。不追求一步到位全自动化，而是识别核心瓶颈、优先覆盖收益最大的环节。

## 一句话总结

> 好的 Skill 不是让 AI 替代人，而是用工程化手段（原子化拆分、机制保障、知识沉淀、状态管理）把 AI 约束在它擅长的"机械执行"区间内，同时在它不擅长的"判断决策"区间留出人工确认节点。
