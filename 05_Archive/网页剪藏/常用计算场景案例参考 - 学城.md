本文讲解在实时数仓场景下常用计算场景下 **聚合、去重、join** 三种算子的使用方法。

## 2.重点算子讲解

### 2.1 聚合

| 分类 | 产出方式 | 适用场景 | 案例 |
| --- | --- | --- | --- |
| **窗口聚合** | - Append Stream - 依赖时间属性 - 窗口数据完整到达后下发结果 | - 每隔一段时间统计结果 	- 过去一分钟异常量 	- 过去一小时成功率 | // 民宿业务，通过滑动窗口，每 30s 计算一次，最近一分钟的客户端异常数量。  ``` xxxxxxxxxx ```  ``` SELECT uuid, level, text, COUNT(1) AS cnt ``` ``` FROM toast ``` ``` WHERE level LIKE '0%’ ``` ``` GROUP BY uuid, level, text, ``` ``` HOP(tp_formatted,INTERVAL '30' SECOND,INTERVAL '1' MINUTE) ``` |
| **全局聚合** | - Retract Stream - 每条数据下发结果 - 依赖状态过期清理 | - 实时更新结果 	- 当前数量 	- 当前最大值 | // 充电宝业务，实时计算每个机柜上充电宝的数量。  ``` xxxxxxxxxx ```  ``` SELECT addr.cabin_id, ``` ``` COUNT(battery_id) battery_cnt ``` ``` FROM addr WHERE switch_status = 1 ``` ``` GROUP BY addr.cabin_id ``` |

  

### 2.2 去重

| 分类 | 产出方式 | 适用场景 | 案例 |
| --- | --- | --- | --- |
| **Count Distinct** | - Retract Stream | - 计数场景下的去重 | // 充电宝业务，实时计算每个机柜上当前正在充电的充电宝数量。业务库一次可能下发多条记录  ``` xxxxxxxxxx ```  ``` SELECT addr.cabin_id, ``` ``` COUNT(DISTINCT IF(switch_status = 1, ``` ``` battery_id, CAST(NULL AS INTEGER))) ``` ``` battery_cnt ``` ``` FROM addr GROUP BY addr.cabin_id ``` |
| **Select Distinct** | - Retract Stream | - 多条重复消息取其中一条 - 一般配合 IN 语句用 | ``` xxxxxxxxxx ```  ``` SELECT DISTINCT addr.battery_id ``` ``` FROM addr ``` |
| **使用聚合函数**  **Last value** | - Retract Stream | - 指定取某一条 	- 最大的一条 	- 最新的一条 | ``` xxxxxxxxxx ```  ``` SELECT LAST_VALUE(cabin_id), ``` ``` LAST_VALUE(battery_id), ``` ``` LAST_VALUE(switch_status) ``` ``` FROM addr ``` |

  

### 2.3 关联

#### 2.3.1 双流 Join

双流join：流和流的Join， 在有限的计算和存储资源下，在资源使用和关联准确率上做取舍

| **分类** | **产出方式** | **适用场景** | **案例** |
| --- | --- | --- | --- |
| **Interval Join** | - Append Stream - 超过等待时间后下发一边为空的内容 | - 不希望重复 - 不希望错误数据下发 - 允许关联不上适当等待 | // 广告业务用券分析  ``` xxxxxxxxxx ```  ``` SELECT * FROM Orders ``` ``` LEFT JOIN Coupon ``` ``` ON Orders.coupon_id = Coupon.id ``` ``` AND Orders.ts BETWEEN ``` ``` Coupon.ts - INTERVAL '30' MINUTE ``` ``` AND Coupon.ts + INTERVAL '30' MINUTE ``` |
| **Regular Join** | - Retract Stream - 直接下发一边为空的内容，关联上再撤回 - 依赖状态过期清理 | - 不允许等待 - 允许数据大量重复 - 下游能根据回撤进行结果修正 | ``` xxxxxxxxxx ```  ``` SET 'flink.exec.state.retention.time.min'= 1800000; ``` ``` SELECT * FROM Orders ``` ``` LEFT JOIN Coupon ``` ``` ON Orders.coupon_id = Coupon.id ``` |

  

#### 2.3.2 流表 Join

- 流数据到达时，进行维表的查询
- 维表存储在支持按 key 查询的存储服务上（例如 Tair 或者 Hbase）

⚠️ 如果维表更新频率较高需要保证维表更新及时，否则需要让流数据延迟一段时间去查询 。

```
xxxxxxxxxx
```

```
SELECT  Orders.id, Dim.name
```
```
FROM Orders AS o
```
```
JOIN Dim FOR SYSTEM_TIME AS OF o.proctime
```
```
ON Orders.id = Dim.id
```

  

  

#### 2.3.2 多流 Join、流表多次 Join

多流Join 和 流表多次Join 的使用方式为将他们顺次接起来，但注意 需要人工调节Join的最优顺序 （因为flink不会像mysql那样做关联顺序优化）

**case1** ：两个LEFT JOIN相连👇

```
xxxxxxxxxx
```

```

```

```

```

```
```

```

```

```
```

```
Product.ts - INTERVAL '30' MINUTE AND Product.ts + INTERVAL '30' MINUTE;
```
