### 项目一：codedoc — 多仓代码资产文档化与跨角色知识问答平台

面向新人上手、QA、PM、架构师在大规模多仓代码下的理解与协作，基于 AST 构建多语言统一代码知识图谱，支撑文档自动生成，自然语言代码问答与跨仓变更影响分析。

1. 多语言代码图谱：Python ast、Java javalang、JS 正则统一成一套节点和边模型；除类、方法、字段外识别 Spring 路由与 IoC bean、FastAPI 与 Flask 路由等框架节点，定义 calls、imports、route_handler、bean_inject 等七类边，可跨 IoC 注入与 HTTP 路由追踪调用链。

2. 意图分流混合检索：问答按意图分语义、关系、全局三链路，分别走向量加全文倒排双路召回再 BGE-reranker 精排、图谱遍历调用方被调用方、GraphRAG 社区摘要。调 ef_search 把向量 recall@20 从 25% 提到 96%；另建可复现评测，top-5 命中 97%、合成 recall@5 85%、关系 recall@20 0.84。

3. 抗幻觉问答与 prompt 调优：grounding prompt 内置可引用符号白名单与事实约束，答完逐个核验引用符号是否真实存在于图谱并打可信度，flask、click、Springboot 实测引用可信度从 6% 提到 96%。

4. 生产级多轮会话记忆：锚点、焦点、笔记等结构化记忆按会话跨请求持久化，乐观锁加 CAS 防并发丢更新，叠加 token 预算对话窗口与溢出轮异步滚动摘要、崩溃可重建；省略主语的追问据此定位上一轮符号并补调用关系，记忆只增量不替代检索，关系追问实测四题全答出真实调用者。

5. Agent 工具与技能层：把代码理解写成 search、context、callers、callees、impact、explore、get_body 等原子工具，注册成一张后端可注入的工具表，经 MCP 给 Claude Code 和 Cursor 挂载，也供 Web 问答和命令行复用，一份实现多处消费。

6. 多仓多 Agent 协同：用户把相关仓关联成业务线，用 LangGraph 编排 Planner 路由、Send 派发 RepoAgent 各跑 ReAct 工具循环、Merger 拼跨仓关系、Synthesiser 汇总，会话记忆单仓多仓共用，单仓走直连检索不绕 Agent。

7. 图谱驱动文档生成：按十七章模板，用节点度数挑核心类和路由作主干，事实层全从图谱抽取不靠 LLM 编造，再从带类型注解的签名生成 PlantUML 架构图、类图和时序图。

8. 异步任务、持久化与生产加固：克隆、索引、文档生成入 PostgreSQL 队列跨四 worker 一致、断点可恢复，代码图谱 JSONB 落库共享，watchdog 增量同步单文件两三秒生效；另做多租户鉴权、索引崩溃续传、checkpoint 恢复、限流降级、webhook 幂等，精准上下文使 token 较 chunk-RAG 省 86%、较整文件省 98%。

### 项目二：devbot — GitHub 智能 PR 评审

研发能效 Agent 平台与 GitHub 智能 PR 评审。面向团队 PR 评审与研发提效场景，通过 MCP 接入 codedoc 图谱获取跨仓变更影响，用 LangGraph 编排多 Critic、单 Agent 与工具调用，覆盖 PR 评审、需求拆解、代码生成与单测生成，并自建评测管道支撑 prompt 迭代。

1. 多 Critic 并行评审与 finding 去噪：四个 Critic 经 LangGraph 并行评分，加权聚合加一票否决定风险，领域纪律 prompt 让风险分从普通 90 分化正交；评测定位已报真因是因 Critic 重复报同一缺陷，跨 Critic 去重加 severity 校准后误报 123 降到 12，定位精度 0.19 升到 0.84，召回 0.86，F1 0.30 升到 0.52。

2. 需求拆解、代码生成与单测生成 Agent：需求文本并行跑范围、风险、验收三维度自动出子任务和验收点；代码生成经 codedoc 语义检索取相似实现与依赖影响子图做精准上下文，替代纯 RAG 大窗口堆砌；单测从 diff 和调用图提取调用链与边界场景，生成测试骨架、mock 与断言。

3. GitHub CI 集成：Webhook 接 PR 的 opened 和 synchronize 事件并做 HMAC-SHA256 校验，支持在 commit 写 review、ask、gen-test 指令触发，结果写回 PR Review 评论和 Commit Status，风险等级供分支保护规则决定是否合入。

4. Prompt 工程化与自建 Eval：prompt 用 registry 做 YAML 版本管理、按 PR id hash 确定性灰度；自建 Eval Harness 覆盖风险准确率、finding F1、ECE 校准、注入抵抗、长度退化等 7 个指标，配 PSI 漂移与 CI 回归门禁，在 52 个真实样本上跑通四 Critic 与 LLM judge，并用 Platt 后置校准把置信度 ECE 从 0.37 降到 0.24。
