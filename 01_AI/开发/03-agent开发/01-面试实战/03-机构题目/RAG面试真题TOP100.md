1. **RAG 全流程**：文档加载 → 分块 → 向量化 → 存储 → 检索 → 重排 → 生成。每个环节的技术选型和优化策略。
2. **GraphRAG vs 传统RAG**：为什么引入图谱、社区发现算法、混合检索策略、适用场景。
3. **召回率优化手段**：Query改写、HyDE、多路召回、混合检索、Rerank、上下文压缩等。
4. **分块策略对比**：固定长度 vs 语义分块 vs 递归分块，overlap设计，chunk_size选择依据。

### 必须有实战经验支撑的

1. **向量数据库选型**：Milvus/Chroma/Qdrant 的适用场景、性能对比、数据规模承载。
2. **PDF/复杂文档解析**：表格提取、图片OCR、多栏布局处理、公式识别。
3. **知识图谱+RAG融合**：Neo4j构建流程、实体抽取、关系查询、与向量检索的配合。
4. **评估体系搭建**：RAGAS指标、端到端评估、人工评测流程。

### 加分项（体现深度）

1. Self-RAG / Agentic RAG / Corrective RAG 等前沿范式。
2. Embedding模型微调（对比学习、负样本构建）。
3. 生产环境的延迟优化与并发方案。
4. RAG系统的可观测性与持续迭代机制。
