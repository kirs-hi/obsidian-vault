## 项目深问清单（第 1 周练熟，第 2 周继续补）

这一节是整个计划的核心。这些题面试 90% 会被问到，而且都绑在你的项目上，你必须答上来。

### A. Agent / LangGraph

1. **你的 ReAct Agent 和教科书 ReAct 有什么区别？**
   - 教科书是 Thought→Action→Observation 纯文本循环；我的是结构化 JSON action，LLM 只输出工具选择
   - 防死循环三道闸：max_iter + 签名去重 + 无新节点早停
   - dossier 工具：一步取某符号的全邻域（callers+callees+impact+explore+body），减少 agent 来回
   - 失败降级：agent 异常或没探到 → 退回确定性 fallback（search 定位 + 全工具档案）

2. **LangGraph 的 StateGraph 和 LangChain AgentExecutor 区别？**
   - StateGraph 是状态机，节点+边+条件边，可以非线性路由、有环、扇出
   - AgentExecutor 是线性 ReAct 循环，没法做"supervisor 决策下一步走哪个 agent"
   - LangGraph 支持 Send 扇出并行、PostgresSaver 检查点、conditional edge 路由

3. **你的两套多 Agent 架构为什么不一样？**
   - `agents/graph.py`：Planner 一次性选仓 + fan_out 并行，适合仓已确定的场景，线性 + 反思环
   - `multi_agent.py`：Supervisor 状态机，每阶段回 supervisor 看进展再决策，支持运行时扩仓
   - 取舍：前者快但固定，后者灵活但贵（多一轮 LLM 决策）

4. **Reflect 反思环怎么触发？为什么只 1 轮？**
   - 触发条件：react 模式 + (LLM 判 insufficient 或 答案含 hedge 词) + rounds < 1
   - 只 1 轮：反思本身要调 LLM，多轮成本高且容易反复横跳；1 轮补检索已能覆盖大部分缺口

5. **PostgresSaver 检查点存什么？怎么续传？**
   - 每个 super-step 把整个 state 落 PG，按 thread_id 索引
   - 续传：用相同 thread_id 再 invoke，从最后一个 checkpoint 恢复 state 继续

### B. 检索 / RAG / 向量

6. **三路混合检索怎么融合的？**
   - 语义（BGE-M3 向量 top-k）+ 全文（倒排 top-k）+ 关系（图遍历 callers/callees）
   - 候选合并后过 BGE-reranker 交叉编码精排，取 top-N 喂 agent
   - 不是简单拼，是 reranker 统一打分排序

7. **为什么用 BGE-M3 不用 OpenAI embedding？**
   - 多语言（问中文、代码英文都能 embed）、长上下文、免费、可本地部署
   - OpenAI 要翻墙+花钱，国内访问慢

8. **reranker 放哪一步？为什么不一开始就用？**
   - 放召回之后：召回要快（HNSW 近邻），reranker 是交叉编码慢，只能对小批量候选做
   - 先粗召回 top-40 → reranker 精排 → 取 top-15

9. **pgvector HNSW 的参数你调过吗？ef_search 是干嘛的？**
   - ef_search 是搜索时的邻居队列大小，越大越准越慢
   - 我设的 ef_search 在 `codedoc_select.py` 里 SET hnsw.ef_search，召回和延迟的平衡点

10. **GraphRAG 和普通 RAG 区别？什么时候用？**
    - 普通 RAG：局部问题（某函数怎么实现），向量召回就够了
    - GraphRAG：全局问题（整体架构/主要子系统），零散召回拼不出全貌
    - 我用正则 `is_global_question` 判断"架构/整体/overview"等关键词路由到 GraphRAG
    - Louvain 划社区 → 每社区 LLM 一句话摘要 → map-reduce 汇总回答

11. **Louvain 为什么选它？不用别的社区算法？**
    - networkx 自带、快、基于模块度优化、适合稀疏图（代码依赖图就是稀疏的）
    - 别的（如 Girvan-Newman）慢，谱聚类要定簇数

### C. 知识图谱 / 解析

12. **跨文件调用怎么连边的？重名怎么办？**
    - 跨文件调用按 Python LEGB（同文件/同类/闭包）作用域消歧
    - 每条边标 confidence（unique/scope）：唯一名直接连，重名按作用域选最可能的
    - 比"仅唯一名才连"召回高很多

13. **9 类节点 8 类边分别是什么？**
    - 节点：模块/类/方法/字段/参数 + 框架节点（Spring 路由/IoC bean、Flask/FastAPI 路由）
    - 边：调用/继承/实现/引用/依赖 + 框架特有边（路由注册/IoC 注入）
    - （具体去 `parser/` 下背熟）

14. **为什么要识别框架节点？**
    - Spring 的 IoC 注入、Flask 的路由注册，调用关系不是显式函数调用，AST 看不出来
    - 不识别就丢了整条 HTTP 路由链和依赖注入链

### D. 抗幻觉 / 评测

15. **groundedness 怎么算？阈值多少？**
    - 答案里提到的符号，去图谱查在不在；存在率 = groundedness
    - < 0.5 判定答案不扎实，触发反思补检索
    - hedge 词（"未找到/无法回答"）也触发

16. **选仓 F1 0.96 怎么测的？GT 哪来的？**
    - 13 道真实微服务多仓问题，人工标注每题该选哪些仓
    - 对比 4 种策略（纯搜/+正向扩/+无脑反向/+改进），改进版（hub 排除+反向编排判断）最优
    - （这是 `select_eval.py`，能把代码讲出来）

17. **ablation 做了什么？**
    - 记忆消融：无记忆 / 仅 anchor / 全套，对比多轮问答效果（`eval/mem_ablation.py`）
    - 选仓策略消融：4 种选仓策略对比

### E. 工程 / 生产

18. **SKIP LOCKED 怎么抢单的？**
    - `UPDATE ... WHERE task_id = (SELECT ... FOR UPDATE SKIP LOCKED LIMIT 1)`
    - 多 worker 并发各锁各的，被锁的跳过，不会阻塞
    - 比 Redis 队列少一个组件，和 PG 数据在一起

19. **任务崩了怎么办？**
    - running 任务带 heartbeat，超时（默认 120-900s）的 `reclaim_orphans` 打回 queued
    - 其他 worker 重新抢单重跑；attempts 超上限标 error

20. **限流怎么做的？为什么不用令牌桶？**
    - PG 固定窗口计数：按 user/IP 维度，每窗口 N 次
    - 不用进程内令牌桶：多 worker 下额度会 ×N，不准
    - 限流自身异常 fail-open（放行），不拖垮主流程

21. **为什么全栈用 PostgreSQL 不拆？**
    - 图谱 JSONB、向量 pgvector、队列 SKIP LOCKED、检查点 PostgresSaver、记忆 CAS，一个库搞定
    - 零额外运维，跨 worker 天然一致
