**项目介绍** ：公司研发团队日常编码中重复性工作多、跨项目上下文切换成本高，为提升工程效率，主导自研终端 AI 编程助手。基于 ReAct 与 Plan Mode 双模式驱动 LLM 自主完成编程任务，五层分层架构支持多模型接入、MCP 工具生态扩展、跨会话记忆和多 Agent 并行协作，已在团队内部推广使用。

**个人职责** ：

-   从 0 到 1 负责整体架构设计与技术选型，确定五层分层架构方案，完成核心模块开发与团队内部推广落地
-   设计并实现多 LLM 供应商接入层，统一 Anthropic、OpenAI 两套流式协议为同一套事件接口，支撑团队按需切换模型
-   负责上下文管理与成本控制方案，设计两层渐进式压缩策略，保障长时间编程会话不中断的同时控制 Token 消耗

**技术亮点** ：

-   **MCP 工具延迟加载** ： 设计 MCP 工具延迟加载机制，百级工具场景下工具描述 Token 占用减少 85%，解决了工具生态膨胀后上下文被挤占的问题
-   **多模型统一接入** ： 抽象统一接入层适配 Anthropic、OpenAI 双协议， **新增模型供应商只需实现一个适配接口** ，团队可按成本和能力灵活切换模型
-   **两层上下文压缩** ： 设计两层渐进式上下文压缩策略， **单次会话可持续数小时不丢失关键上下文** ，Token 成本较无压缩方案显著降低
-   **五层权限拦截** ： 设计五层纵深权限拦截模型， **任一层拒绝即终止操作** ，保障 Agent 在全自动模式下安全执行，未出现过越权操作事故
-   **自动记忆提取** ： 设计跨会话自动记忆提取机制， **跨会话知识持续累积，同类错误纠正一次后团队全员自动规避，新会话自动继承项目上下文无需重复交代**
-   **多 Agent 并行协作** ： 设计多 Agent 并行协作与文件级隔离机制， **大型重构任务处理效率成倍提升**
