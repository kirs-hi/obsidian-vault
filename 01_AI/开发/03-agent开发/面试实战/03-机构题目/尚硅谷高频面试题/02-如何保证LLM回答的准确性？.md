# 第2章 如何保证LLM回答的准确性？

> 来源：尚硅谷大模型智能体之高频面试题 V2.1.5

---

2.  如何保证LLM回答的准确性？

多层保障机制：

- 检索质量：多路召回+重排，确保检索到最相关的⽂档

- Prompt设计：明确要求\"基于事实回答，不编造信息\"

- temperature=0：使⽤确定性输出，避免随机性导致的错误

- 引⽤来源：要求LLM引⽤参考资料编号，便于溯源验证

  1.  如何保证系统的高可用？

      1.  模型单例：BGE-M3、qwen-flash等使用单例模式，避免重复加载

      2.  批量操作：Milvus 批量插入，减少网络往返

      3.  容错处理：LLM JSON 解析失败时进行清洗和容错

      4.  降级策略：商品名确认失败时返回澄清选项，而非直接报错

      5.  资源清理：SSE 队列在 finally 块中清理，防止内存泄漏

  1.  扩展思考类

      1.  如果让你重新设计这个系统，你会做哪些改进？

          1.  缓存层：添加 Redis 缓存热门问题和答案

          2.  对话摘要：长对话使用 LLM 生成摘要，而非简单裁剪

          3.  分布式部署：使用 Kubernetes 实现水平扩展

          4.  可观测性：集成 OpenTelemetry 实现分布式追踪

          5.  A/B 测试：对比不同检索策略的效果

          6.  向量数据库分区：按商品名分区，加速检索

      2.  如何评估RAG系统的效果？

主要评估指标：

- 召回率（Recall）：检索到相关文档的比例

- 准确率（Precision）：检索结果中相关文档的比例

- MRR（Mean Reciprocal Rank）：第一个相关结果的排名倒数

- Answer Quality：人工评估答案的准确性、完整性、可读性

- 响应时间：端到端延迟，包括检索+生成
