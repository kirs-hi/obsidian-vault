> 调研平台：智联招聘（zhaopin.com）浏览器实地抓取

## 一、市场概况

2026年 AI 行业从"百模大战"进入"应用落地与商业化变现"阶段。据多个行业报告统计，全球 AI 人才缺口已突破 500 万，中国市场占比约 30%，人工智能岗位供需比约 1:10。脉脉平台数据显示，2026 年以来新发 AI 岗位中算法与研发岗占比不足 15%，意味着近 85% 的 AI 岗位核心要求并非算法研究，而是应用落地能力。

AI 应用开发（AI Application Development）和 AI Agent 开发（AI Agent Development）已成为技术招聘市场最热门的方向。不同于传统算法工程师需要深厚的数学功底和论文积累，这两类岗位更强调工程落地能力——把大模型（LLM）集成到实际业务场景中。

**但需要注意的是，行业报告中的"平均薪资"数据存在严重偏高**。例如多篇分析文章声称"北京 AI Agent 平均月薪 40K"，但这个数字主要被字节、蚂蚁、百度等大厂的高薪岗位拉高。从智联招聘真实在招数据来看，北京市场上大量 AI Agent / AI 应用开发岗位的实际薪资集中在 1.2 万-2.5 万区间，中位数约 1.8 万/月。

## 三、岗位分类与职责差异

### 3.1 AI 应用开发工程师

核心定位：将大模型能力集成到业务系统中，构建可落地的 AI 产品。

典型职责：

- 基于 LLM API 开发企业级 AI 应用（智能客服、文档问答、数据分析助手等）
- RAG 系统搭建：知识库构建、向量化处理、检索策略优化
- Prompt 工程迭代与优化
- AI 中台建设、技术架构设计
- ML 工具和平台维护

### 3.2 AI Agent 开发工程师

核心定位：构建具备自主推理、工具调用、任务规划能力的智能体系统。

典型职责：

- 智能体核心开发：意图识别、任务规划（Planning）、记忆管理、工具调用
- Agent 工作流编排：状态机设计、意图路由、多步骤任务执行
- MCP（Model Context Protocol）工具开发与接入
- 多智能体（Multi-Agent）协作系统设计
- 基于 LangChain/LangGraph/Dify/AutoGen 等框架开发 Agent
- 知识库 + RAG 检索增强与 Agent 结合

### 3.3 真实市场中的第三类：AI 训练师/评测

这是从真实数据中发现的一个重要品类，分析文章通常不提，但实际岗位量不少：

- 核心内容：AI Agent 的标注、评测、效果测试、Prompt 优化
- 薪资范围：1-2 万/月
- 典型公司：外企德科、软通动力等（外包性质居多）
- 门槛较低，但成长空间有限

### 3.4 两者的关键差异

| 维度 | AI 应用开发 | AI Agent 开发 |
|------|-----------|--------------|
| 侧重点 | LLM 集成 + RAG + API 封装 | 自主推理 + 工具调用 + 任务编排 |
| 复杂度 | 单轮/多轮对话为主 | 多步骤自主决策链路 |
| 技术深度 | Prompt + RAG + 结构化输出 | Planning + Memory + Tool Use + Multi-Agent |
| 薪资溢价 | 基准线 | 比 AI 应用开发再高 15-30% |
| 面试考察 | RAG pipeline + 系统设计 | Agent Loop 手写 + 架构设计 + 工程化 |

## 四、核心技能栈分析

### 4.1 编程语言

**Python 是绝对核心**。从智联招聘真实数据来看，约 70% 的岗位明确要求 Python，其次是 Java（约 25%），少量要求 Go/C++。面试中，面试官更青睐手写 Agent Loop 来展示对底层机制的理解，而非单纯依赖 LangChain 等框架。

### 4.2 框架与工具（按真实 JD 出现频率排序）

| 技术 | 出现频率 | 说明 |
|------|---------|------|
| Python | 最高 | 几乎所有岗位的硬性要求 |
| LangChain / LangGraph | 高 | Agent 开发首选框架 |
| RAG + 向量数据库 | 高 | Milvus/Chroma/pgvector，知识库检索核心 |
| LLM / 大模型 | 高 | 通用能力要求 |
| Agent / 智能体 | 高 | 岗位核心定义 |
| Function Calling / Tool Use | 中高 | 工具调用是 Agent 的基础能力 |
| MCP（Model Context Protocol） | 中 | 2026 年新增热门 |
| Dify / RAGFlow | 中 | 低代码 Agent 编排平台 |
| Flask / Django | 中 | Python Web 框架 |
| MySQL / Redis | 中 | 基础数据存储 |
| Docker / K8s | 中低 | 工程化部署能力 |
| PyTorch / TensorFlow | 中低 | 偏算法方向才要求 |

### 4.3 核心能力要求

按重要程度排序：

1. **RAG 检索增强生成**：知识库构建、向量化、检索策略、重排序（Rerank）、噪声过滤。这是几乎所有 AI 应用岗位的必备技能。

2. **Agent 工作流编排**：意图路由、状态机、工具调用链、错误处理与重试。理解 ReAct / Reflection / Plan-and-Execute 等 Agent 设计模式。

3. **Prompt 工程**：结构化输出、Few-shot、Chain-of-Thought、上下文裁剪。不仅是使用，还要理解优化策略。

4. **LLM API 调用与集成**：OpenAI / DeepSeek / Qwen 等模型 API 的使用，流式输出处理，Token 成本优化。

5. **工程化能力**：微服务拆分、容器化部署、日志监控、性能优化。这是区分"会调 API"和"能上生产"的关键。

6. **系统设计能力**：能设计完整的 AI 应用架构，包括数据流、服务拆分、缓存策略、并发处理。

## 六、北京真实市场全景分析

以下统计基于智联招聘实际抓取的约 120 个岗位数据。

### 6.1 学历要求分布

| 学历要求 | 占比 | 说明 |
|---------|------|------|
| 本科 | ~65% | 绝对主流 |
| 硕士 | ~12% | 主要是大厂和头部 AI 公司 |
| 大专 | ~5% | 外包/小公司 |
| 学历不限 | ~3% | 极少数 |
| 博士 | ~1% | 博士后岗位 |
| 经验不限（含实习） | ~14% | 实习为主 |

**结论：二本本科学历可以投递约 70% 以上的岗位。** 只有约 12% 的岗位明确要求硕士，这些需要跳过。

### 6.2 经验要求分布

| 经验要求 | 占比 | 说明 |
|---------|------|------|
| 1-3年 | ~35% | 很多中小公司标这个，实际也接受 3-5 年经验的人 |
| 3-5年 | ~30% | **你的经验正好落在这里** |
| 5-10年 | ~10% | 高级/负责人岗位 |
| 经验不限 | ~15% | 多为实习或门槛较低岗位 |
| 在校/实习 | ~10% | 排除 |

**结论：你 5 年经验可以投递约 80% 的岗位**（1-3年、3-5年、经验不限都可以投）。

## 八、求职策略建议

### 8.1 目标公司优先级

**第一梯队（最推荐）：** 中型上市公司和 B 轮以上创业公司。学历门槛相对宽松，薪资 1.5-2.5 万，有一定稳定性。代表：昆仑联通、千方科技、和利时、新港海岸、中科软。

**第四梯队（过渡）：** 外包/驻场。薪资 1-2 万，但可以快速积累 AI 项目经验，后续跳到直招岗位。代表：软通动力、外企德科。
