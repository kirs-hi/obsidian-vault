# 真题墙 — 产品调研 · 需求分析 · PRD

## 一、产品调研

### 1.1 产品概述

**真题墙**（zhentiqiang.com）是一款面向考研数学备考人群的在线刷题平台，由个人开发者 KingSum（B站同名账号）独立运营。核心定位是"考研数学历年真题 + B站名师视频汇编 + 刷题进度追踪"三合一工具。

产品形态：Web 网页（PC 为主），无 App。技术栈：Flask 后端 + Tabler UI 前端 + CDN 图片服务。商业模式：会员订阅制（¥19/月、¥49/季、¥159/年）。

### 1.2 市场规模

考研市场数据（2025年）：

- 全国考研报名人数：**388万**（2024年为438万，略有下降但仍为超大体量）
- 考研数学是必考科目（数一/数二/数三覆盖大部分理工经管专业），估计 **250万+** 考生需要备考数学
- 刷题工具类产品客单价：通常 ¥20-200/年
- 潜在市场规模（TAM）：250万 × ¥100（平均客单价）= **2.5亿/年**
- 个人开发者可触达市场（SAM）：按 1% 转化率，约 2.5万付费用户 × ¥100 = **250万/年**

独立开发者典型收入参考：此类垂直刷题工具月收入 **¥3K-30K** 属正常水平，做得好可达 ¥5-10万/月。

### 1.3 用户画像

| 维度   | 描述                       |
| ---- | ------------------------ |
| 年龄   | 20-26岁，大三大四学生 + 二战/三战考生  |
| 痛点   | 真题散落各处、无系统刷题记录、不知道考点分布规律 |
| 使用场景 | 电脑前刷题（对照纸质试卷看解析）、手机查进度   |
| 付费意愿 | 中等（¥10-20月可接受，毕竟是备考刚需）   |
| 获客渠道 | B站视频引流（教解题思路）、考研社群口碑传播   |
| 核心需求 | 按考点分类真题、追踪做题进度、错题本       |

### 1.4 竞品分析

| 竞品 | 定位 | 优势 | 劣势 | 价格 |
|------|------|------|------|------|
| **真题墙** | 考研数学刷题 | 界面优雅、考点清晰、视频关联 | 仅数学、功能较简单 | ¥19-159/年 |
| **粉笔考研** | 全科考研 | 全科覆盖、AI批改 | 体量大、不够垂直 | ¥200+/年 |
| **研木答** | 考研数学题库 | 题库全、有解析 | UI一般、体验普通 | ¥99-199/年 |
| **考研帮/文都** | 综合考研平台 | 品牌力强、课程丰富 | 臃肿、刷题体验差 | ¥500+/年 |
| **小猿搜题类** | 通用拍照搜题 | 覆盖广 | 不专注考研、无系统性 | 免费+增值 |

**真题墙的差异化优势**：极度垂直（只做考研数学）、个人维护成本低、B站人设IP引流、UI体验好（Tabler暗色主题非常适合长时间刷题）。

### 1.5 商业模式分析

收入结构：纯会员订阅，无广告、无课程售卖。

成本结构（估算）：
- 服务器：¥100-300/月（轻量云服务器即可）
- 域名+CDN：¥50/月
- 人力：0（个人开发者）
- B站内容制作：个人时间投入

**优势**：几乎零边际成本，一旦建好题库，后续只需维护和每年更新新真题。毛利率接近 100%。

**风险**：版权问题（真题图片来源）、个人精力有限、市场集中度低（容易被大厂平台覆盖）。

## 二、需求分析

### 2.1 核心功能拆解

基于对线上产品的逆向分析，功能模块如下：

**P0 — MVP 必须**：
1. **历年真题浏览**：按年份/卷型（数一二三）查看试卷列表
2. **题目图片展示**：每道题以图片形式展示（含题干+选项）
3. **做题标记系统**：熟练（绿）/ 不熟（黄）/ 错题（红）三态标记
4. **考点目录树**：三级结构（高数→一元微分→导数计算），点击可筛选
5. **刷题进度统计**：总进度百分比、各试卷进度
6. **用户登录**：手机号+验证码注册/登录

**P1 — 核心体验**：
7. **考点分值统计图表**：柱状图展示各考点历年总分值
8. **错题本**：汇总所有标记为错题的题目
9. **刷题笔记**：每题可添加个人笔记
10. **会员系统**：付费解锁全部年份试卷
11. **B站视频关联**：每题挂载对应讲解视频链接

**P2 — 增长功能**：
12. **试卷模板下载**：PDF 格式空白试卷（引流）
13. **评论留言**：用户交流区
14. **数据看板**：全站用户做题统计（社交激励）
15. **暗色/亮色主题切换**

**P3 — 未来扩展**：
16. **移动端适配**（或小程序）
17. **AI 智能出题**（基于薄弱考点）
18. **多科目扩展**（英语/政治）
19. **答题计时器 + 模拟考试模式**

### 2.2 用户旅程

```
首次访问 → 浏览首页（看到考点树+试卷列表）
    ↓
点击某年试卷 → 弹出登录 → 手机号注册
    ↓
进入做题页 → 逐题浏览图片 → 标记状态
    ↓
做完一套 → 查看进度 → 错题本回顾
    ↓
免费额度用完 → 弹出会员购买页 → 付费
    ↓
持续使用 → 每年续费
```

### 2.3 MVP 范围定义

**第一版上线目标**（1-2周开发周期）：

- 覆盖 2009-2025 年数学一/二/三全部真题（约 51套 × 22题 = 1100+ 题）
- 完整的做题 + 标记 + 进度追踪闭环
- 考点目录树 + 分值统计图表
- 简单的会员付费（可先用手动发卡模式过渡）
- B站视频链接嵌入

**不做的事**：
- 不做 App（纯 Web 即可）
- 不做 AI 功能（MVP 阶段不需要）
- 不做社区/评论（分散精力）
- 不做多科目（先打透数学）

### 3.2 功能需求详细设计

#### F1: 首页（试卷列表页）

**页面结构**：
- 顶部导航栏：Logo + 功能Tab（历年真题/考点目录/数据统计/视频课程/评论留言）+ 登录按钮
- 筛选栏：卷型下拉选择（数学一/二/三）+ 显示难度/笔记开关 + 公告栏入口
- 左侧：考点目录树（三级折叠）
- 右侧上：试卷卡片网格（年份 + 题目数 + 进度条）
- 右侧中：刷题进度环形图
- 右侧下：各考点历年总分值柱状图

**交互规则**：
- 切换卷型 → 刷新试卷列表和图表
- 点击试卷卡片 → 跳转到做题页
- 点击考点叶子节点 → 右侧显示该考点下的所有题目列表
- 未登录可浏览，登录后显示进度数据

#### F2: 做题页（试卷详情页）

**页面结构**：
- 顶部：试卷标题 + 卷型标签 + 进度 + 前后卷切换
- 左侧：题目导航网格（4列，数字按钮，颜色标识状态）
- 中间：题目内容区
  - 题型/题号/分值/考点/难度星级
  - 题目图片（白底，含公式和选项）
  - 选择题交互选项（ABCD）
- 底部：上一题/下一题 + 标记按钮（熟练/不熟/错题）
- 右侧：答案（折叠）+ 我的笔记（文本框）+ 同考点关联题目列表

**交互规则**：
- 左右箭头键/AD键切换题目
- 数字键 1/2/3 快速标记
- 标记后导航按钮变色、进度实时更新
- 笔记自动保存到后端
- 点击关联题目可跨试卷跳转

#### F3: 用户系统

**登录方式**：手机号 + 短信验证码（一键注册登录）

**用户数据**：
- 基本信息：手机号、昵称、头像
- 会员状态：等级、到期时间
- 做题记录：每题的标记状态、笔记、最后更新时间

#### F4: 会员系统

**套餐设计**：
| 套餐 | 价格 | 时长 | 原价 | 折扣 |
|------|------|------|------|------|
| 月卡 | ¥19 | 30天 | ¥29 | 6.6折 |
| 季卡 | ¥49 | 90天 | ¥87 | 5.6折 |
| 年卡 | ¥159 | 365天 | ¥348 | 4.6折 |

**付费墙策略**：
- 免费用户：可浏览全部题目、查看最近3年真题
- 付费会员：全部年份解锁 + 不限量笔记 + 错题本导出 + 去除水印

**支付接入**：微信支付（小程序支付/H5支付）

#### F5: 数据统计

**图表内容**：
- 各考点历年总分值柱状图（帮助用户判断重点章节）
- 个人刷题进度环形图
- 错题分布热力图（按考点维度）

### 3.3 技术方案

#### 架构设计

```
┌─────────────────────────────────────────┐
│              前端 (Static)               │
│  Tabler UI + Chart.js + Vanilla JS      │
│  CDN: jsDelivr (CSS/JS库)               │
└─────────────┬───────────────────────────┘
              │ HTTP API
┌─────────────▼───────────────────────────┐
│              后端 (Node.js)              │
│  Express + JWT + Cookie Auth            │
│  Routes: auth/papers/topics/progress    │
└─────────────┬───────────────────────────┘
              │
┌─────────────▼───────────────────────────┐
│            数据库 (SQLite)               │
│  users / papers / questions /           │
│  topics / user_progress                 │
└─────────────────────────────────────────┘
              │
┌─────────────▼───────────────────────────┐
│         题目图片 (CDN/OSS)               │
│  按 paper_id + question_no 命名          │
│  格式: SVG/PNG, 白底黑字                 │
└─────────────────────────────────────────┘
```

#### 数据模型

**users 表**：id, phone, password, nickname, avatar, member_level, member_expire, created_at

**topics 表**：id, parent_id, name, sort_order, subject

**papers 表**：id, year, paper_type, title, total_questions

**questions 表**：id, paper_id, topic_id, question_no, question_type(choice/fill/solve), content, answer, difficulty(1-5), score, image_url, video_url

**user_progress 表**：id, user_id, question_id, status(done/unsure/wrong), is_wrong, note, updated_at

#### 部署方案

- 服务器：轻量云（腾讯云/阿里云 2C4G，¥100/月）
- 域名：已有 zhentiqiang.com
- HTTPS：Let's Encrypt 免费证书
- 图片：OSS + CDN（或 GitHub Pages 免费托管）
- 监控：UptimeRobot 免费监控

## 四、可复用要点（给独立开发者的启示）

1. **选垂直赛道**：不要做大而全，做"一个人能维护"的极度垂直产品
2. **IP + 产品联动**：B站/小红书人设引流，产品承接付费
3. **技术极简**：一个人不要上微服务架构，SQLite + 静态文件就够了
4. **付费墙精准切割**：让用户先体验到价值，再收费
5. **复利型产品**：题库一旦建好，每年只需增量更新，边际成本趋近于零
6. **暗色主题**：刷题/学习类产品天然适合深色 UI，减少视觉疲劳
7. **键盘快捷键**：刷题效率工具必须支持，提升用户粘性

## 六、用户量估算（2026年5月）

### 6.1 核心数据点

| 渠道 | 数据 |
|------|------|
| B站"真题墙官方" | 粉丝 2,388，主推视频播放 7.9万，收藏 5,718 |
| 抖音爆款引流 | 苦瓜学长/考研的秘密推荐视频，单条点赞 99.8万 |
| 抖音搜索热度 | "真题墙收费"、"真题墙会员"、"真题墙使用教程"均为独立热搜词条 |
| 实时在线（镜像站） | "现在有1人正在陪你一起刷题" |
| 多平台口碑 | 抖音多KOL自发推荐、百度贴吧/知乎有用户讨论、QQ频道有使用帖 |
| 模式验证 | 已出现"控制工程真题墙"（kongzhi.feitent.com）仿站 |

### 6.2 估算逻辑

**抖音爆款效应**：99.8万赞视频，按5%点赞率推算播放量约2000万。即使0.5%观众访问网站，也带来10万次访问。这是真题墙最大的流量来源（远超B站）。

**B站引流**：7.9万播放 × 5%转化 ≈ 4,000访问。收藏5,718说明有较强"稍后使用"意愿。

**实时在线个位数**：说明日活不会超过几千（考研工具使用场景集中且单次使用时间短）。

**抖音有独立搜索热词**（"真题墙收费"、"真题墙会员权益"）：说明有足够多的用户在讨论付费问题，至少千人级付费用户。

### 6.3 用户量估算

| 指标 | 保守 | 中位（最可能） | 乐观 |
|------|------|----------------|------|
| 累计注册用户 | 2-3万 | 5-8万 | 15万+ |
| 付费会员（累计） | 2,000 | 5,000-8,000 | 1.5万 |
| 月活（考研季9-12月） | 5,000 | 1-3万 | 5万 |
| 月活（非考研季） | 500 | 2,000-5,000 | 1万 |
| 日活（考研季） | 500-1,000 | 2,000-5,000 | 1万 |

### 6.4 收入估算

按中位估计：5,000-8,000付费用户 × 平均客单价¥80（季卡¥49~年卡¥159之间）

| 指标 | 估算 |
|------|------|
| 累计总收入 | ¥40-64万 |
| 年收入 | ¥15-30万 |
| 月收入（考研季高峰） | ¥2-5万 |
| 月收入（非考研季） | ¥0.3-1万 |

### 6.5 结论

真题墙是一个**累计注册5-8万、付费5000-8000人、年收入15-30万**的个人项目。对独立开发者而言属于"甜蜜点"：足够养活自己、不需要团队、边际成本几乎为零、每年只需更新当年新真题。主要流量来自抖音而非B站，抖音爆款视频是核心增长杠杆。
