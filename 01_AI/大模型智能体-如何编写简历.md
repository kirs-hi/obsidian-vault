## 一、Word 简历制作

### 1.1 简历注意事项

1. 全篇字体不要超过三种，可以用 **"微软雅黑"、"宋体"、"楷体"**
2. 注意排版（尽量上下左右对称）、单倍行距或1.5倍行距、标点符号中文
3. 行末标点要统一（统一加或统一不加）
4. **正文建议5号字**
5. 在左上角装订，不要在其它地方装订
6. 简历要保持整洁，不要折，不要手写改动，最好放在文件袋里
7. 递交简历时要双手递上，正面要朝向面试官，方便其阅读

### 1.2 电子简历文档名称

```
xxx_大模型开发工程师_xx年_简历.docx
xxx_智能体开发工程师_xx年_简历.docx
```

说明：boss投递时使用pdf格式。

### 1.3 简历名称

> 【图片内容】简历标题示例：展示了一份简历的头部格式，包含姓名、职位方向、技术栈关键词等排版

- title不要写"个人简历"，标准写法应包含姓名+目标岗位+核心技术栈

### 1.4 个人信息

#### 基本情况

- **必填的：** 姓名、年龄、性别、工作年限、联系电话、邮箱（不要用qq邮箱）
- **选填的：** 学历、专业、籍贯、学校（好的写上）
  - 是一本的、211的、985的、双一流的，标识清楚
  - 民办本科，甚至更低的，就不建议写学校名了。可以写本科（统招）
- **加分项：** 个人网站、github地址、发表论文、技术博客地址、公众号地址（如果粉丝量大，加星多，很牛）
- **不要填的：** 身高、体重、血型、星座、照片？不要！

### 2.2 获奖情况

1. 英语四、六级，具备良好的英语阅读能力（也可放到个人基本情况里）

5. 大学或工作期间发表过大模型相关论文
6. 项目获得了技术研发部门的好评并获得公司颁发的10万元项目奖金

## 四、专业技能

### 4.1 用词修饰

- **最高级：** 精通
- **第2级：** 熟练掌握/熟练使用
- **第3级：** 深入理解
- **第4级：** 熟悉
- **第5级：** 了解

### 4.2 公式

```
（修饰词） xxx技术，及 xxxx（相关深层次的理论知识）
```

### 4.3 具体技能点

#### 开发工具

- **编程语言：** 熟练掌握 Python 及 NumPy、Pandas 数据处理；熟悉 Java/C++ 等开发语言，可适配多场景开发。
- **模型工具链：** 熟练使用 Hugging Face Transformers 调用预训练模型，熟悉 PEFT 与 Accelerate 等训练方式。
- **工程化工具：** 熟练掌握 Docker 封装隔离环境、K8S 编排容器。

#### Python 基础

- **语法与数据结构：** 熟练使用 Python 语法，精通列表、字典等数据结构的高效操作。
- **文件操作与异常处理：** 能读写多种格式文件，熟练掌握异常处理，保障程序稳健运行。
- **模块与包管理：** 熟练使用 pip、conda、uv 等模块包管理工具，维护稳健的运行环境。

#### 数据分析

- **数据处理：** 熟练使用 Pandas 进行高效的数据清洗、转换工作，构建可靠的数据集。
- **数据可视化：** 熟练 Matplotlib、Seaborn 制作各类可视化图表。
- **统计分析：** 熟悉描述性统计、回归等方法，能用 SciPy 开展统计计算分析。
- **数据库操作：** 熟练掌握 SQL 操作 MySQL/PostgreSQL，了解连接池优化数据库访问性能。

#### 大模型框架原理

- **Transformer 体系：** 深解 Transformer 核心组件，熟悉 BERT、GPT 等变体结构特点。
- **模型架构与优化：** 了解模型架构对性能的影响，掌握 AdamW 等优化器与梯度裁剪等技巧。
- **预训练与微调：** 理解大模型预训练通用逻辑，掌握适配特定任务的大模型微调方法。
- 熟悉大模型的缩放定律，了解参数量、数据量与性能之间的正比例关系，持续关注大模型领域最新研究趋势。

#### 大模型微调

- **微调方法与策略：** 掌握全参数/PEFT微调，能够依据任务、资源情况，选择部分层冻结、预热、学习率衰减等策略。
- **数据准备与处理：** 熟悉数据清洗、数据增强流程，构建高质量的微调数据集。
- **微调过程与优化：** 熟悉大模型微调原理，能合理调整训练策略避免模型训练过拟合/欠拟合。
- **模型评估与部署：** 用 BLEU 等模型性能评估指标，了解 vllm 部署大模型的方法。

#### RAG

- **RAG 框架原理：** 深解检索增强生成逻辑，熟悉检索器与生成器协同工作机制。
- **检索技术应用：** 熟用向量/VSM 检索技术，掌握 TF-IDF、BERT Embeddings 文本向量化。
- **RAG 实践与优化：** 熟悉 Advance RAG、Agentic RAG，rerank、rrf 方法，有效优化生成文本质量。
- **RAG 应用场景：** 了解智能问答等应用场景，能够设计适配需求的 RAG 解决方案。

#### 大模型应用

**LangChain：**

- 熟练使用 LangChain 框架及其组件 memory、tools、model/io、chain 等，深入理解 RAG 技术及 Agent 原理和应用
- 熟练使用 LangChain 快速构建基于企业级私域数据的智能问答及 Agent 应用
- 熟练使用 LangChain 框架，集成多种大语言模型以构建高级 NLP 应用，如问答系统、文本生成和对话系统
- 基于 LangChain 框架，设计和开发公司特定场景应用的 Agent，包括多轮对话系统和复杂任务链条的开发等
- 熟练使用 LangChain 框架、自定义工具、prompt+few-shot 提示工程，构建问答系统、文本生成和对话系统
- 熟悉 Agent 设计方法，包括 React、FunctionCall，嵌入记忆，反思、规划、行动能力构建；能根据业务需求构建测试数据集，对提示词进行优化

**LangGraph：**

- 熟练使用 LangGraph 框架及核心组件（StateGraph、节点/边、Checkpoint等），深入理解图结构 Agent 原理、状态流转及多 Agent 协作，掌握非线性工作流编排技术
- 熟练使用 LangGraph 构建企业级私域数据智能问答及多 Agent 应用，依托状态共享能力实现任务闭环管控与持久化运行
- 熟练使用 LangGraph 集成大语言模型与外部工具，构建多步骤推理、闭环对话、多角色协作 Agent 等高级 NLP 应用
- 基于 LangGraph 设计开发企业场景复杂 Agent 及工作流，含多轮闭环对话、多 Agent 协作及复杂任务编排，适配多业务场景
- 熟练使用 LangGraph、自定义节点/工具及提示工程，结合状态管理，构建高效可控的 NLP 应用，保障工作流稳定可调试
- 熟悉 LangGraph Agent 设计方法，掌握状态定义、条件路由、Plan-and-Execute 架构，具备反思优化能力，可优化节点逻辑与提示词提升执行效率

**Coze、Dify：**

- 使用 Dify+本地大模型快速构建企业级智能体应用
- 熟练使用 Coze、Dify 平台，私有化部署 Dify + 本地大模型快速构建企业级智能体应用
- 熟练使用 Dify 平台搭建公司各种业务场景的工作流和 agent 智能体，并且做 api 调用和优化应用等
- 熟练运用 Coze、Dify 平台开发智能体，涵盖提示词工程、RAG 知识库搭建、Workflow、多种工具的使用

**大模型部署：**

- 掌握本地/云平台部署 Dify、Coze 及 Ollama、XInference 平台下 DeepSeek/Qwen/GPT、Embedding Model、Rerank Model 的部署
- 掌握阿里云/腾讯云的 Docker 及 Dify、Coze 部署，AutoDL 平台 Ollama、XInference 及各种嵌入模型、重排序模型部署

1. 熟悉 Python，熟练掌握 Python 基础语法、基本数据类型、文件读写。熟练使用迭代器、生成器、装饰器、多线程、多进程进行编程。

15. 熟悉 RAG 索引检索生成流程，熟悉文档解析，多种文本分块方式，多种文档索引方式。熟悉 RAG 的检索前与检索后处理，如查询扩展、查询转换、HyDE、混合检索、rrf 重排序等。
16. 熟悉多种 RAG 指标和能力的评估，熟悉 RAGAS 评估框架的使用。熟悉 GraphRAG 的工作流程。
17. 熟悉 Vit 架构、扩散模型、Unet 网络、CLIP 架构等。

## 五、工作经历

### 5.1 公司选择

1. 知名度较高的公司，不推荐选择
2. 员工人数推荐在500-1500左右。100人以下不推荐
3. 建议优选xxx公司的地方分公司，便于第三方背调时方便沟通
4. 收集公司基本信息，主要关注公司发展史、主营业务、是否存在母公司等信息
5. 联系方式通常是无法联系到人事及开发部门的，但最好打个电话确认下
6. 牢记公司地址，了解周边环境，标志性设施、交通设施需牢记
7. 确定租房位置、通勤情况
8. 公司有智能化产品时，将工作经历与智能化产品适当结合

- 模型蒸馏优化策略更新

#### 示例十一：海淘跨境智能客服

**项目背景：** 海外动作为融合了纽约第五大道众多高端百货的跨境直购中台，对接超过 6000 个品牌与 500万+ SKU，业务面临的核心挑战在于：非标品的海淘咨询复杂，用户在购买欧美服饰时，面临严重的"国内外尺码标准不一"痛点（如 US/UK/EU 码换算）。传统的 FAQ 库难以覆盖海量且实时变动的海外专柜库存信息，导致人工客服在处理"尺码推荐"和"海外库存咨询"时压力巨大。

**核心技术：** Qwen2.5-VL、Neo4j、LlamaFactory、GraphRAG、LangGraph、Agent、vllm、PostgreSQL+

**个人职责：**
- 搭建海淘知识图谱 GraphRAG：使用 bge-m3 将欧美品牌尺码对照表、海淘案例指南和跨境物流关税规则文档向量化，构建品牌品类尺码库图谱，覆盖尺码换算与推荐逻辑
- 商品实体与库存同步：使用 UIE 模型抽取商品标题中的核心实体，编写 Cypher 语句将实体与 MySQL 中实时库存及海外官方 SKU 相关数据同步到 Neo4j
- 多模态增强微调：基于 LlamaFactory 对 Qwen2.5-VL-7B-Instruct 进行 SFT，重点优化模型对商品海外实拍图细节的视觉理解能力
- Multi-Agent 架构：基于 LangGraph 设计 Multi-Agent 系统，每个子 agent 专注处理各自细分领域问题，负责商品信息咨询、订单服务模块的实现
- 基于 PostgreSQL 实现长期记忆和短期记忆，并实现用户 profile，用以针对不同用户产生个性化回答
- 基于 MCP 的 Streamable Http 协议，将 GraphRAG 检索功能封装为 MCP 服务并提供给大模型，通过 Function Calling 进行检索

**项目成果：**
- 意图识别 F1 达到 0.89，Cypher 正确率达到 0.85+
- 将转人工率从 76% 降低到 50% 以下
- 商品及库存查询速度大大提升，复杂查询生成的首 token 响应时长缩短

#### 示例十二：数策-Text-to-SQL 系统

**项目背景：** 公司聚合了美国各大百货商场的数据，涉及海量跨境订单与海外供应商库存表，数据结构复杂且分散。业务人员（买手、跨境运营、选品经理）需要实时掌握动销率、品牌售磬率、跨境物流时效等关键指标。主要痛点：业务人员不懂 SQL，IT 部门排期紧张；涉及多表关联，通用模型难以选择合适的业务数据表，生成 SQL 准确率极低。

**核心技术：** LangGraph、RAG、PGVector、jieba 分词、FastAPI、DeepSeek、sqlglot、向量检索

**个人职责：**
- Text-to-SQL 工作流设计：基于 LangGraph 设计多节点工作流，包含指标拆解（如将"热销"拆解为"销量/海外库存深度"）、上下文提取、Schema 检索、SQL 生成与校验
- 业务元数据增强 RAG：搭建 RAG 系统，使用 pgvector 存储业务术语与表结构的向量映射，结合 jieba 分词提取业务关键词，实现对"跨境电商特有业务口径"的精准匹配
- 多模型协同架构：集成 DeepSeek 等模型，针对复杂的多表关联查询选择最优模型
- 双库架构与指标服务：设计 MySQL（业务数据）+ PostgreSQL（向量数据）架构，开发 Metrics 服务，预定义高频指标 API

**项目成果：**
- 业务自助查询：自然语言转 SQL 准确率 >85%，买手与运营人员可直接提问获取数据
- 复杂场景覆盖：支持 100+业务表（含线上订单、线下库存、会员积分）的智能关联
- 实时决策支持：预定义指标查询 <500ms，提升了特卖活动的复盘效率
