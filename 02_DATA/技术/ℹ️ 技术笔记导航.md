## 使用说明

这个 base 有四个视图，点上方标签页切换：

- **全部笔记**：所有技术笔记表格，按最近修改排序。
- **按目录浏览**：以卡片形式按子目录（flink / 数据治理 / Spark 等）分组。
- **最近更新**：最近改过的 15 篇，快速回到正在写的东西。
- **待补全属性**：列出还没有 `tags` 的笔记。把这些笔记补上 frontmatter 后，分类和检索会更准。

## 为什么要补 frontmatter

Bases 靠笔记顶部的属性（frontmatter）来筛选和分类。目前 47 篇技术笔记里约一半没有属性，所以「待补全」视图存在。建议新笔记统一用这套最小属性：

```yaml
---
title: 笔记标题
created: 2026-06-15
tags:
  - flink        # 或 spark / 数据治理 / 数仓建模 等
status: 草稿      # 草稿 / 完成 / 长期维护
---
```

补全后可以再给 base 加「按 status 看板」「按标签筛选」等视图。建议用 Templater 做个新建技术笔记的模板，自动套这套属性，避免手敲。
