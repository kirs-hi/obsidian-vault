# SLA治理调研

## 背景
- SLA期望与实际落差大
  - 业务期望提前但产出滞后
  - 缺乏量化评估标准
- 上游链路长且复杂
  - 跨部门依赖多
  - 链路层级深（最高26层）
- 末端任务逼近SLA无缓冲
  - 留给优化的时间窗口极小
  - 任何上游延迟直接穿透
- 不可控因素影响大
  - 资源争抢
  - 上游数据延迟
- 治理需体系化前置
  - 被动救火转主动预防
  - 需要系统性方法论

## 难点
- 难点一：复杂网状链路
  - 最高26层
    - 层级间依赖交叉
    - 难以线性梳理
  - 上百任务
    - 任务间存在隐式依赖
    - 变更影响面难评估
  - 瓶颈难识别
    - 关键路径动态变化
    - 需要自动化工具辅助
- 难点二：长期治理落地
  - 缺少抓手
    - 无统一治理平台
    - 数据分散难聚合
  - 缺少机制
    - 无定期review流程
    - 权责不清晰
- 难点三：研发能力差异
  - 优化建议难落地
    - 不同团队技术栈差异
    - 需要提供具体可执行方案

## 解法
- 核心链路识别
  - 从末端任务回溯
    - 逆向遍历DAG
    - 标记所有祖先节点
  - 同层级最晚完成任务
    - 取每层耗时最长的任务
    - 即为当层瓶颈
  - 形成核心链路集合
    - 瓶颈任务串联成链
    - 聚焦关键路径优化
  - 判定有效优化点
    - 优化收益 > 阈值才纳入
    - 避免无效投入
- 三阶段拆解
  - 数据采集
    - 跨部门沟通
      - 推动上游提前产出
      - 建立SLA协议
    - 提前上游依赖
      - 调整调度时间
      - 减少等待空窗
    - COW升级HIDI
      - 架构升级提升写入效率
      - 减少锁竞争
    - 就绪时间优化
      - 合理设置ready time
      - 避免空等资源
  - 数仓计算
    - Spark参数优化
      - executor数量/内存调整
      - shuffle分区数优化
    - SQL逻辑优化
      - 减少不必要的JOIN
      - 谓词下推/分区裁剪
    - 链路精简
      - 去除冗余中间表
      - 合并可复用逻辑
    - 增量替代全量
      - 降低计算量级
      - 缩短执行时间
  - 数据下发
    - Doris模型优化
      - 合理选择数据模型
      - 优化物化视图
    - Load策略调整
      - 批量导入优化
      - 并发度调整
    - 引擎迁移Kylin到Doris
      - 统一OLAP引擎
      - 降低运维复杂度
    - 增量导入
      - 减少全量刷新
      - 提升下发效率

## 长线运营机制
- 事前
  - 风险SLA识别
    - 基于历史数据预测风险
    - 自动标记高危任务
  - 每周告警
    - 定期推送风险报告
    - 提前介入处理
- 事中
  - SLA治理大盘
    - 应用分析
      - 按应用维度聚合SLA达成率
      - 识别高频违约应用
    - 链路分析
      - 可视化关键路径
      - 实时标注瓶颈节点
    - 任务分析
      - 单任务耗时趋势
      - 异常波动检测
    - 人员分析
      - 按负责人统计治理进展
      - 推动责任到人
  - 优化建议输出
    - 自动生成优化方案
    - 关联SparkUI指标
- 事后
  - 进展追踪
    - 优化项闭环管理
    - 定期复盘效果
  - 影响权重
    - 量化每项优化的SLA收益
    - 优先级排序
  - 权责划分
    - 明确上下游责任边界
    - 建立escalation机制

## 生产视角问题治理
- 数据采集
  - 就绪时间不合理
    - ready time设置过晚
    - 导致下游空等
  - 架构升级COW到HIDI
    - 提升并发写入性能
    - 减少数据延迟
- 数仓计算
  - 高耗时任务
    - 识别Top N慢任务
    - 针对性优化SQL/参数
  - 全量生产任务
    - 评估增量改造可行性
    - 逐步迁移至增量模式
  - 上游阻塞任务
    - 分析阻塞根因
    - 推动上游优化或解耦
  - 大文件任务
    - 文件合并策略调整
    - 减少IO开销
  - 小文件任务
    - [[小文件合并]]
    - 调整输出分区数
  - 维表瓶颈任务
    - MapJoin替代ReduceJoin
    - 维表预加载/广播
  - [[数据倾斜实践|数据倾斜]]
    - 加盐打散热点key
    - 两阶段聚合
  - 文本存储改ORC
    - 列式存储压缩率高
    - 查询性能提升
- 数据下发
  - OLAP引擎选型
    - 根据场景选Doris/CK/Kylin
    - 统一收敛引擎数量
  - 全局字典密集
    - 字典膨胀导致构建慢
    - 优化编码策略
  - 全量下发问题
    - 改为增量下发
    - 减少数据传输量

## 资源视角治理
- 夜间资源利用率低
  - 并发断层
    - 任务调度时间不均匀
    - 存在大段空闲期
  - 冷热分离
    - 低频任务移至低峰期
    - 释放高峰期资源
  - 任务合并
    - 相似逻辑合并执行
    - 减少重复计算
  - 依赖调整
    - 优化DAG拓扑
    - 提升并行度
- Yarn调度滞后
  - Pending排查
    - 分析资源等待原因
    - 区分队列满/资源不足
  - Cantor调度优化
    - 调度策略参数调优
    - 减少调度延迟
- 弹性队列不合理
  - 夜间比例调整
    - 提高夜间弹性配额
    - 匹配实际负载曲线

## 工具与能力建设
- 元数据获取链路
  - 自动采集任务元信息
  - 构建任务画像
- 风险策略识别
  - 规则引擎自动判定
  - 多维度风险评分
- SparkUI指标分析
  - Shuffle数据量
    - 监控shuffle read/write
    - 识别数据膨胀
  - 倾斜判断
    - Task耗时方差分析
    - 自动定位倾斜key
  - Stage耗时
    - 各Stage耗时占比
    - 识别慢Stage根因
  - 资源Pending比例
    - Pending时间占比监控
    - 触发资源扩容建议
- 自动化分析平台建设
  - 一键诊断任务瓶颈
  - 自动生成优化建议
  - 治理效果可视化追踪
