# 采购数据与智能运营平台｜简历版

## 1. 项目定位

**2025.01 ～ 至今｜美团-采购数据与智能运营平台｜数仓开发**

采购业务覆盖境内外 **22 个系统、300+ 原始表**，存在主数据不统一、链路割裂、经营分析依赖人工取数、跨系统风险事后发现等问题。项目以采购全流程数据为核心，建设“**统一数据底座 + 实时运营闭环 + AI 数据消费**”能力。

---

## 2. 核心职责与成果

### 一、统一数据底座：解决数据分散与口径不一

- 制定需求、建模、指标、运维四类数仓规范，以“不登记口径不上看板、不评审不接需求”推动治理机制落地。
- 完成 **22 个系统、300+ 表**的盘点、分级接入与标准化处理，DWD 明细层覆盖度 **82.3%**。
- 基于 ODS-DWD-DWS-ADS 分层及总线矩阵建设采购域模型，统一品类、供应商、组织、时间等一致性维度；完成三级品类体系及跨系统供应商编码映射。

### 二、可信数据服务：解决质量与复用问题

- 建立完整性、准确性、一致性、唯一性、及时性等 **7 类 DQC** 规则体系，核心链路 DQC 覆盖率 **100%**；按强弱规则分级阻断和告警。
- 建设指标门户及采购数据应用，推动存量应用 **100%** 切换至标准模型，统一指标口径并降低人工对数成本。

### 三、实时采购运营：解决跨系统风险发现滞后

- 围绕“**合同 → PO 订单行 → 履约/收货 → 验收 → 付款**”构建订单行累计快照，以 `po_code + po_line_id` 聚合跨系统当前状态。
- 设计 Flink CDC + Flink + Paimon 实时链路：CDC 接入业务增量事件，Flink 完成状态关联、乱序处理及超时规则计算，Paimon 主键表维护订单行最新快照。
- 支撑未验收订单、合同临期履约、合同额度穿透等风险识别，并回流采购工作台形成“**待办聚合 → 责任人触达 → 原系统处置 → 状态自动关单**”闭环。

### 四、AI 数据消费：解决长尾分析依赖人工的问题

- 设计采购取数分析 Skill，覆盖需求澄清、权限预检、多维取数、洞察分析与报告生成。
- 采用“**意图理解 → 结构化分析蓝图 → SQL 模板执行**”模式替代裸 NL2SQL；通过行列级权限前置、分层降级和全链路审计，保证数据安全、结果可控与过程可追溯。

## 4. 90 秒面试话术

这是一个采购数据与智能运营平台项目。采购链路覆盖境内外 22 个系统，合同、订单、履约、验收、付款数据分散，既有主数据和指标口径不统一的问题，也存在跨系统风险只能 T+1 甚至事后发现的问题。

我负责从零建设统一数据底座，先完成 22 个系统、300 多张表的盘点和分级接入，DWD 覆盖度达到 82.3%。模型侧通过总线矩阵统一了品类、供应商、组织、时间等一致性维度，重点解决了跨境内外系统的品类层级和供应商编码不一致问题。同时建立 7 类 DQC 规则，核心链路覆盖率达到 100%，并推动存量应用全部切换到标准模型。

在此基础上，项目向上建设两类消费能力。第一类是实时运营：围绕合同到付款链路，以订单行为粒度建设累计快照；通过 Flink CDC 接入事件、Flink 做状态关联和超时规则、Paimon 维护最新快照，识别未验收订单、合同临期履约等风险，回流采购工作台形成待办到关单的闭环。第二类是 AI 消费：设计采购 Data Agent，用结构化蓝图和 SQL 模板代替裸 NL2SQL，并通过权限前置、降级和审计保障结果可控。

项目核心价值是把采购数据从分散的事后报表，升级为统一可信、可实时行动、可被 AI 消费的数据资产。
