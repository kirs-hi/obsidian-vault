# AI 简历生成 Prompt 模板

> 来源：拾穗数据工作室（微信：shisuidata）
> 核心原则：让 AI 帮你表达真实，不让 AI 替你编造能力。

---

## 使用前准备三份材料

在让 AI 改简历之前，先准备：

1. **原始经历材料**：项目背景、你做了什么、工具、数据、结果、团队分工。
2. **目标岗位 JD**：职责、要求、加分项、业务场景、工具栈。
3. **个人边界说明**：哪些你真实做过，哪些只是了解，哪些不能写。

如果没有这些材料，AI 改出来的简历会像一件租来的西装，看着合身，一坐下就崩线。

---

## 一套完整工作流

```
JD拆解 → 经历盘点 → 项目重写 → 关键词匹配 → 简历体检 → 面试追问反推 → 定稿投递
```

---

## Prompt 1：JD 拆解

```
你是一位数据行业招聘顾问。请帮我拆解下面这个岗位JD。

要求：
1. 提炼岗位的真实职责，不要只复述原文。
2. 将要求分为：硬技能、业务能力、AI相关能力、软技能、隐含要求。
3. 判断这个岗位更偏：分析/运营/开发/治理/数据产品/AI数据岗位中的哪一种。
4. 给出简历应该重点呈现的5类证据。
5. 输出我应该避免写的空话。

岗位JD：
[粘贴JD]
```

## Prompt 2：经历盘点

```
你是一位严谨的简历教练。请根据我提供的经历，帮我盘点可写进简历的项目证据。

要求：
1. 不要编造任何经历。
2. 先补齐项目所属公司/组织、项目起止时间、我当时的岗位或角色。
3. 将经历按隐性 STAR 拆成：S-业务场景、T-目标任务、A-关键行动、R-最终结果。
4. 在 A-关键行动里继续拆出：使用工具、AI参与环节、人的判断、人工校验方式。
5. 如果某一项缺失，请标注"待补充"，不要替我编。
6. 判断哪些经历适合放在目标岗位简历首页。
7. 输出一个"证据强度评分"：强/中/弱，并说明原因。

我的经历：
[粘贴经历]

目标岗位：
[粘贴岗位或JD]
```

## Prompt 3：简历项目重写

```
你是一位数据行业招聘顾问。请把下面的项目经历改写成适合投递[目标岗位]的简历表达。

写作规则：
1. 底层必须遵循隐性 STAR：业务场景、目标任务、关键行动、最终结果。
2. 表达上使用"业务问题-方法与工具-AI协作-人工校验-量化结果-沉淀资产"的结构，不要机械堆砌。
3. 每条项目描述控制在2-3行，语言专业、具体、克制。
4. 不要编造数字，没有数据的地方用"可补充指标"标注。
5. 不要滥用"赋能、闭环、抓手、智能化"等空话。
6. 生成3个版本：保守版、专业版、强匹配JD版。
7. 最后列出面试官可能追问的5个问题。

目标岗位JD：
[粘贴JD]
```

## Prompt 4：AI 经历安全改写

```
请把下面这段经历中的AI相关表达改得更真实、更可信。

要求：
1. 区分"我真实完成的工作"和"AI辅助完成的环节"。
2. 将AI能力写成具体流程，例如：假设生成、SQL解释、报告初稿、文案变体、字段说明、错误排查。
3. 必须加入人工校验或风险控制方式。
4. 删除夸大表达，例如"完全自动化""精准预测""显著提升"但没有证据的说法。
5. 给出最终可放进简历的版本。

原始内容：
[粘贴内容]
```

## Prompt 5：关键词匹配但不堆词

```
请对比我的简历和目标JD，找出关键词匹配差距。

要求：
1. 提取JD中的关键能力词，并按重要程度排序。
2. 判断我的简历中是否已有真实证据支持这些关键词。
3. 如果有证据，请建议自然嵌入方式。
4. 如果没有证据，请标注"不能硬写"，并建议我用作品集或补充项目弥补。
5. 输出一个更适合该岗位的简历技能模块。

目标JD：
[粘贴JD]

我的简历：
[粘贴简历]
```

## Prompt 6：简历体检

```
请作为严格的面试官，审查我的简历。

请重点检查：
1. 是否存在AI能力过度包装。
2. 是否存在结果数字无法解释。
3. 是否存在只写职责、不写成果。
4. 是否存在工具堆砌，没有业务问题。
5. 是否存在项目描述太像AI生成文本。
6. 是否存在面试中容易被追问穿的问题。

请输出：
- 风险等级：高/中/低
- 问题清单
- 每个问题的修改建议
- 更稳妥的改写版本

我的简历：
[粘贴简历]
```

## Prompt 7：一岗一版生成

```
请基于我的真实经历，为下面这个岗位生成一版定制简历。

硬性要求：
1. 不编造经历和数字。
2. 保留我已经做过的项目，不要凭空新增项目。
3. 根据JD调整项目顺序和表达重点。
4. 输出：专业摘要、核心技能、项目经历、可补充材料建议。
5. 标注所有"需要我补充确认"的地方。
6. 语言要像真实候选人，不要像宣传稿。

我的经历材料：
[粘贴经历]
```

---

## Prompt 8：面试追问反推简历

```
请基于下面这版简历，模拟面试官追问。

要求：
1. 每个项目至少提出5个追问。
2. 追问覆盖：业务背景、技术细节、AI使用环节、校验方式、结果口径、失败复盘。
3. 判断哪些简历描述我可能讲不清楚。
4. 对讲不清楚的部分，建议删减或降级表达。

简历内容：
[粘贴简历]
```
