


    
**基础数据层建设**

- **规范制定**：从0到1制定数仓全流程规范，覆盖需求对接、模型设计、指标管理和数据运维四个环节，与业务、产品达成共识并推动落地。
    
- **数据整合与建模**：摸排境内外10+系统的数据现状，识别300+张原始表，完成5个采购系统及17个外部系统的数据入湖与清洗，明细层覆盖度82.3%。重点对采购品类、供应商两个核心维度做标准化处理与模型设计，替换了此前依赖手工维护的低质量主数据。

- **质量监控**：对核心生产链路配置DQC规则，覆盖率100%。下游数据问题从依赖人工对数切换为监控告警，问题发现和响应时间明显缩短。

*实时化资产层建设 
- **全链路风险T+0预警**：为**解决采购履约风险（如未验收订单滞留、超期付款）发现滞后至T+1的痛点，主导推进了基于 Flink + Paimon + Doris 的实时数仓建设**。设计并实现了“合同-订单-验收-付款”全链路实时宽表，成功将风险发现窗口从“事后报表”前移至“事中预警”，关键业务风险实现T+0响应。

智能化服务层建设
 **多端数据产品统一**：**从0构建了统一的指标与维度体系，并以此为基础打造了管理后台与BI看板**，彻底解决了存量应用“数据口径不一”的历史顽疾，实现100%数据源切换，为业务提供了稳定、一致的多维分析底座。

- **Data Agent 决策辅助**：针对采购经理高频、多维的取数分析需求，**主导设计并落地了基于大模型的“采购分析Skill”**。通过内置**意图澄清、权限预检、多维度取数（品类/供应商等）与降级策略**，自动生成HTML可视化报告，将原本需要数小时的多方沟通与取数过程压缩至分钟级，大幅提升了决策效率。




**项目背景**
随着企业数据智能平台（HR/CAP/采购/办公/行政）业务数据的快速增长，原有数仓体系面临严峻挑战，严重影响数据生产效率、数据服务质量、管理层决策效率及下游业务系统的稳定性。2023 年对标行业及公司优秀团队，引入字节 & 外卖 0987 价值评估体系，确保团队质量交付追上内外优秀团队。
**个人职责**
**1. 元数据体系建设：**
在数仓存在"找数难、价值难量化、研发成本高"的背景下，负责搭建元数据仓库与数据治理指标体系，设计业务/技术/管理三域模型及分层架构，落地血缘、DQC、SLA、权限等核心模块，实现数据资产可视化与治理数字化，显著提升找数效率与问题定位效率，支撑团队效率与质量双提升。
**2. 指标一致性：**
指标命名、定义、取数来源、链路加工一致的占比。面对数据表 6000+、数据产品指标口径频繁不一致的问题，负责指标一致性治理，统一语义、生产与查询口径并推动工具化落地，指标一致率由 89.1% 提升到 97%。
**3. SLA 准点率：**
T+1 生产的 SLA 服务准点产出的占比。从局部单点优化升级为全链路结构性治理，搭建了"观测—提取—归因—长效机制"闭环体系，实现了 T+1 生产的 SLA 服务提前 2 小时达成 99%、SLA 服务准点产出占比 99%。
**4. Doris 集群稳定性：**
数据 OLAP 查询部分的 TP95 耗时。建设了面向多租户、高并发场景的 Doris 治理看板，通过精细化的资源计量与管控体系，落地全生命周期治理：
- 事前：建设 SQL 变更风险卡控机制，自动拦截高危 DDL/DML 操作。
- 事中：实施基于负载的自适应限流与大查询熔断策略，防止单点故障扩散。
- 事后：建立慢查询归因分析与红黑榜晾晒机制，推动研发侧代码质量提升。
实现了在业务高速增长下的"稳态运行"与"降本增效"。
**5. 需求承接率：**
通过研发流程诊断与价值场景挖掘，基于 NoCode 将研发常用提效工具与文档平台化，并完成 SQL Copilot 能力调研与团队推广，赋能数据开发提效。需求承接平均交付时长由 6.88pd 缩短至 5.5pd，开发效率提升 20%。
