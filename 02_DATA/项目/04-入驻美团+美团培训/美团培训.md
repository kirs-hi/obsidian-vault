# 美团培训数仓建设与学习效果归因｜简历项目描述

**2024.06 ～ 至今｜美团-商户数字化发展·美团培训数仓建设｜数仓开发**

美团培训是生活服务业人才数字化发展平台，累计培训超 7000 万人次、覆盖 455 城、服务 1400 万在线商家、1.5 万+ 课程 / 99 学院，与合作招聘、开店指南同属商户数字化发展板块、共用基础日志数据。业务只有完课/未完课结果、学习过程数据缺失，埋点漏埋/错埋/同名不同义，培训效果无法归因、课程内容优化无从下手。负责从 0 到 1 建设培训统一数仓，提供学习旅程归因、指标服务与受控 AI 分析能力。

- 针对培训只有完课/未完课结果、学习过程数据缺失、埋点漏埋/错埋/同名不同义的问题，从 0 设计学习行为埋点 schema（观看/暂停/回看/完课/流失）并建事件级宽表，推"控·治·制"把控埋点质量，把完课率从结果指标拆成可归因的过程指标；流量事实表分区日增量 **>200w**，埋点异常发现前置到 T+1，沉淀学习行为 schema 与埋点验收流程。

- 针对流失定位只到课程级、内容优化无从下手的问题，建设课程—课节—知识点三级内容维度，把流失归因从课程级下沉到知识点级，直接驱动课程内容迭代；复用合作招聘菜单/模块/运营位配置维表范式扩展到内容侧，沉淀三级内容维度与知识点级流失归因模型。

- 针对完课/有效学习/留存口径争议、培训效果无法归因的问题，用 OSM+UJM 定指标判断标准，建"报名→学习→考核→认证→就业"学习旅程漏斗与低/中/高粘性分层，做就业终态反向归因——这是采购（付款）和入驻（签约）都没有的长周期终态归因；沉淀指标体系、旅程漏斗与就业归因模型。

- 针对课程推荐/流失干预依赖人工的问题，建课程推荐 + 流失预测 AI（用在用户侧而非采购的取数侧）与 Flink 行为流流失预警（而非 CDC 事务监控）；AI 仅做意图解析/流失预判，SQL 与干预策略经模板约束和人工抽检，沉淀推荐/流失模型与行为流预警链路。**可补充指标：**流失预警触达及时率、课程推荐 CTR。

**项目收益：**

- **行为数据底座**：从 0 设计学习行为埋点 schema + 事件级宽表，流量日增 **>200w**，完课率从结果指标拆成可归因过程指标。
- **学习旅程可归因**：建报名→学习→考核→认证→就业漏斗 + 生命周期分层，培训效果从"只有完课结果"升级为"过程可归因 + 就业可反哺"。
- **指标口径统一**：OSM+UJM 统一核心指标口径，解决"什么算完课"定义争议；埋点异常从用数时发现前置到 T+1。
- **研发效能**：需求响应周期从 **5 天缩短至 1.5 天**，新商家 30 天辅导 **100% 触达**。
- **决策升级**：上线课程推荐 + 流失预测，运营从人工推荐/事后复盘转为 AI 推荐 + 流失 T+0 预警；沉淀六类可复用资产。

## 面试概述（90 秒）

起点是美团培训只有完课/未完课结果、学习过程数据缺失，埋点漏埋/错埋/同名不同义，培训效果无法归因。和采购不同——采购的数据"已存在但散"，矛盾是主数据口径；培训的数据"根本没采到"，矛盾是数据生产方式。

顺序是先造数据再归因效果：从 0 设计学习行为埋点 schema + 事件级宽表并推控·治·制；建课程—课节—知识点三级内容维度把流失下沉到知识点级；用 OSM+UJM 定指标判断标准，建报名→学习→考核→认证→就业旅程漏斗做就业终态反向归因（采购/入驻终态都是一次性，培训是长周期就业）；课程推荐 + 流失预测 AI 用在用户侧（非取数 NL2SQL），Flink 接行为流做流失预警（非 CDC 事务）。AI 仅做意图解析/流失预判，SQL 与干预策略经模板约束和人工抽检。沉淀六类资产：学习行为 schema、三级内容维度、指标体系、旅程漏斗、推荐/流失模型、就业归因模型。
